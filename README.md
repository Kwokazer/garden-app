# Garden - платформа для садоводов и любителей растений

## О проекте

Garden - современная веб-платформа для садоводов и любителей растений, которая позволяет делиться опытом, находить информацию о растениях и покупать всё необходимое для садоводства.

### Основные функции

- **Система аутентификации и авторизации**:
  - Регистрация пользователей с подтверждением по email
  - Вход через логин/пароль и OAuth (Google, VK, Yandex)
  - JWT токены для безопасной аутентификации
  - Управление ролями и разрешениями

- **Персональный профиль пользователя**:
  - Управление личными данными
  - Просмотр статистики активности
  - Управление коллекцией растений
  - История заказов и отзывов

- **Каталог растений**:
  - Просмотр доступных для покупки растений
  - Фильтрация и поиск
  - Детальная информация о каждом растении

## Технический стек

### Фронтенд
- Vue.js 3 (Composition API)
- Vue Router для навигации
- PrimeVue для UI компонентов
- Tailwind CSS для стилизации
- JWT для аутентификации

### Бэкенд
- FastAPI (Python)
- SQLAlchemy ORM
- PostgreSQL для хранения данных
- Redis для кеширования
- Alembic для миграций
- JWT для токенов аутентификации
- SMTP для отправки email

### Инфраструктура
- Docker и Docker Compose для контейнеризации
- Nginx в качестве прокси-сервера
- Python 3.12+
- Node.js 18+

## Переменные окружения

### Настройка окружения

Проект использует переменные окружения для настройки различных компонентов приложения. Для начала работы:

#### Бэкенд

1. Скопируйте файл-пример переменных окружения:
   ```bash
   cp backend/.env.example backend/.env
   ```

2. Отредактируйте файл `backend/.env`, указав ваши значения для:
   - Подключения к базе данных PostgreSQL
   - Подключения к Redis
   - Секретного ключа JWT (для безопасности используйте уникальный ключ)
   - OAuth провайдеров (если требуется)
   - Настроек SMTP для отправки писем (если требуется)
   - Настроек хранилища файлов

#### Фронтенд

1. Скопируйте файл-пример переменных окружения:
   ```bash
   cp frontend/.env.example frontend/.env
   ```

2. Отредактируйте файл `frontend/.env`, указав ваши значения для:
   - URL API бэкенда
   - Настроек OAuth (если требуется)
   - Других специфичных для фронтенда настроек

### Важные переменные окружения

#### Бэкенд

| Переменная | Описание | Пример значения |
|------------|----------|----------------|
| `POSTGRES_*` | Настройки подключения к PostgreSQL | `POSTGRES_HOST=localhost` |
| `REDIS_*` | Настройки подключения к Redis | `REDIS_HOST=localhost` |
| `JWT_SECRET_KEY` | Секретный ключ для JWT (важно для безопасности) | `openssl rand -hex 32` |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | Время жизни токена доступа | `30` |
| `OAUTH_*` | Настройки OAuth провайдеров | `GOOGLE_CLIENT_ID=your_id` |
| `EMAIL_*` | Настройки SMTP для отправки почты | `EMAIL_HOST=smtp.example.com` |

#### Фронтенд

| Переменная | Описание | Пример значения |
|------------|----------|----------------|
| `VITE_API_BASE_URL` | URL API бэкенда | `http://localhost:8000/api/v1` |
| `VITE_GOOGLE_CLIENT_ID` | ID клиента Google OAuth (если используется) | `your_client_id` |

### Безопасность

- Никогда не коммитьте файлы `.env` в репозиторий
- Используйте разные значения для разработки и продакшена
- Регулярно меняйте секретные ключи в продакшен-окружении
- Ограничивайте права доступа к файлам `.env`

## Запуск проекта

### С использованием Docker Compose (рекомендуется)

1. Убедитесь, что у вас установлены Docker и Docker Compose
2. Настройте переменные окружения (см. выше)
3. Запустите все сервисы:

```bash
docker compose up -d
```

4. Доступ к приложению:
   - Фронтенд: http://localhost
   - API бэкенда: http://localhost/api/v1

### Локальное развертывание для разработки

#### Бэкенд

1. Создайте и активируйте виртуальное окружение Python:
```bash
cd backend
python -m venv venv
source venv/bin/activate  # для Linux/Mac
venv\Scripts\activate  # для Windows
```

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Примените миграции:
```bash
python -m scripts.migrate
```

4. Запустите сервер разработки:
```bash
uvicorn app.main:app --reload
```

#### Фронтенд

1. Установите зависимости:
```bash
cd frontend
npm install
```

2. Запустите сервер разработки:
```bash
npm run dev
```

## Разработка

### Структура проекта

#### Бэкенд (clean architecture)
- `app/api` - API эндпоинты и маршруты
- `app/application` - бизнес-логика и сервисы
- `app/domain` - модели и схемы данных
- `app/infrastructure` - внешние сервисы (БД, Redis, Email)

#### Фронтенд (feature-based)
- `src/features` - функциональные модули приложения
- `src/shared` - общие компоненты
- `src/utils` - утилиты и вспомогательные функции
- `src/api` - взаимодействие с API
- `src/router` - маршрутизация
