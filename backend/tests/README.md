# Тесты Garden API

Этот каталог содержит асинхронные тесты для Garden API, написанные с использованием pytest и pytest-asyncio.

## Структура тестов

### Основные тестовые файлы:

1. **`test_auth.py`** - Тесты аутентификации и регистрации
   - Регистрация пользователя с подтверждением email
   - Валидация данных регистрации
   - Подтверждение email по токену
   - Вход в систему

2. **`test_plants.py`** - Тесты работы с растениями
   - Получение информации о растении по ID
   - Получение списка растений с пагинацией
   - Фильтрация и поиск растений
   - Обработка ошибок (растение не найдено)

3. **`test_webinars.py`** - Тесты работы с вебинарами
   - Создание вебинаров (админ и plant_expert)
   - Получение вебинаров где пользователь является хостом
   - Подключение пользователей к вебинарам
   - Проверка прав доступа

4. **`conftest.py`** - Конфигурация pytest и фикстуры
   - Настройка тестовой базы данных
   - Создание тестовых пользователей
   - HTTP клиент для тестирования
   - Фикстуры авторизации

## Установка зависимостей

```bash
# Установка основных зависимостей
pip install -r requirements.txt

# Установка дополнительных зависимостей для тестирования
pip install -r requirements-test.txt
```

## Запуск тестов

### Запуск всех тестов:
```bash
pytest
```

### Запуск конкретного файла тестов:
```bash
pytest tests/test_auth.py
pytest tests/test_plants.py
pytest tests/test_webinars.py
```

### Запуск конкретного теста:
```bash
pytest tests/test_auth.py::TestUserRegistration::test_user_registration_with_email_verification
```

### Запуск с подробным выводом:
```bash
pytest -v
```

### Запуск с покрытием кода:
```bash
pytest --cov=app --cov-report=html
```

### Запуск только быстрых тестов:
```bash
pytest -m "not slow"
```

## Переменные окружения для тестов

Тесты используют SQLite в памяти для изоляции от основной базы данных. Никаких дополнительных переменных окружения не требуется.

## Фикстуры

### Пользователи:
- `admin_user` - Администратор с ролью "admin"
- `plant_expert_user` - Эксперт с ролью "plant_expert"  
- `regular_user` - Обычный пользователь без специальных ролей

### Авторизация:
- `auth_headers_admin` - Заголовки авторизации для администратора
- `auth_headers_expert` - Заголовки авторизации для эксперта
- `auth_headers_user` - Заголовки авторизации для обычного пользователя

### Данные:
- `test_plant` - Тестовое растение
- `db_session` - Сессия тестовой базы данных
- `client` - HTTP клиент для тестирования

## Примечания

1. **Изоляция тестов**: Каждый тест выполняется в отдельной транзакции, которая откатывается после завершения теста.

2. **Мокирование**: Email сервис мокируется для предотвращения отправки реальных писем во время тестов.

3. **Асинхронность**: Все тесты асинхронные и используют pytest-asyncio для корректной работы с async/await.

4. **База данных**: Используется SQLite в памяти для быстрого выполнения тестов без влияния на основную базу данных.

5. **Покрытие**: Тесты покрывают основные сценарии использования API, включая успешные случаи и обработку ошибок.
