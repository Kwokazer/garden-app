# Сервисный слой приложения

Этот документ описывает организацию сервисного слоя приложения после реорганизации.

## Общий подход к организации сервисов

Все сервисы следуют единому подходу:

1. **Именование файлов**: snake_case с суффиксом _service.py (например, user_service.py)
2. **Именование классов**: PascalCase с суффиксом Service (например, UserService)
3. **Именование методов**:
   - Для операций: действие_объект (register_user, update_plant)
   - Для получения данных: get_* (get_user, get_plants)
   - Для CRUD-операций: get_*, create_*, update_*, delete_*
   - Приватные вспомогательные методы: _snake_case (_validate_email, _hash_password)

## Структура сервисов

Сервисы разделены по доменным областям:

1. **Базовые сервисы**:
   - `BaseService` - общий базовый класс для всех сервисов
   - `AuthService` - аутентификация и авторизация
   - `UserService` - управление пользователями

2. **Доменные сервисы**:
   - `PlantService` - управление растениями
   - `PlantCategoryService` - управление категориями растений
   - `ClimateZoneService` - управление климатическими зонами
   - `QAService` - управление вопросами и ответами
   - `PlantSearchService` - поиск и фильтрация растений

## Обзор сервисов

### AuthService

Управляет аутентификацией и авторизацией пользователей, включая:
- Регистрацию пользователей
- Вход в систему
- Выход из системы
- Управление JWT токенами
- Проверку прав доступа

### UserService

Управляет пользователями, включая:
- Получение данных пользователя
- Обновление профиля
- Управление ролями
- Управление настройками пользователя

### PlantService 

Управляет растениями, включая:
- Создание, чтение, обновление и удаление растений
- Управление фотографиями растений
- Связь растений с категориями и климатическими зонами

### PlantCategoryService

Управляет категориями растений, включая:
- Создание, чтение, обновление и удаление категорий
- Получение растений по категории

### ClimateZoneService

Управляет климатическими зонами, включая:
- Создание, чтение, обновление и удаление зон
- Получение растений по климатическим зонам

### QAService

Управляет вопросами и ответами, включая:
- Создание, чтение, обновление и удаление вопросов
- Управление ответами на вопросы
- Голосование за вопросы и ответы

### PlantSearchService

Управляет поиском и фильтрацией растений, включая:
- Фильтрацию по различным параметрам
- Полнотекстовый поиск
- Расширенные опции фильтрации

## Изменения, выполненные в ходе реорганизации

1. **Переименовано**:
   - `SearchService` -> `PlantSearchService` для уточнения назначения

2. **Разделено**:
   - `plant_service.py` на три отдельных файла:
     - `plant_service.py` (PlantService)
     - `plant_category_service.py` (PlantCategoryService)
     - `climate_zone_service.py` (ClimateZoneService)

3. **Обновлены имена методов** для единообразия в соответствии с соглашениями

4. **Переименованы сущности инфраструктурного слоя**:
   - `RedisService` -> `RedisCache` для ясного разделения сервисов и кэш-компонентов

5. **Обновлены сигнатуры методов**:
   - Переход с `redis_service` -> `redis_cache` во всех зависимостях

6. **Создан `__init__.py`** для удобного импорта сервисов
